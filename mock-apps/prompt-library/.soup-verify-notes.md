## QA findings I'd expect on this wave

Blocking rules (per `verify.md` §3): any failing test → BLOCK; any critical security → BLOCK; ≥3 critical correctness → BLOCK; coverage <70% → NEEDS_ATTENTION.

  - LOW (style): `useListPrompts.ts` exposes a custom return type that duplicates TanStack's `UseQueryResult`. Arguably a non-issue; don't over-polish.
  - MEDIUM (testing): only 1 of 8 plan steps has a passing impl in this walkthrough; coverage will read as NEEDS_ATTENTION until T2-T6 land. Expected given scope limits.
  - LOW (docstring): `0001_initial.py` has module docstring but `upgrade`/`downgrade` are terse. Acceptable; fn names are self-explanatory.

- **security-scanner (likely findings):**
  - 0 findings. No secrets, no raw SQL string-concat in app code (DDL is in migrations only per `rules/postgres §3`), no eval/exec.
