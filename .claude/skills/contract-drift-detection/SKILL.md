# Contract Drift Detection

## Overview

Cross-stack contracts — OpenAPI ↔ TypeScript types, `.proto` ↔ gRPC clients,
SQL schema ↔ ORM model, shared zod schemas copy-pasted between `frontend/` and
`backend/` — break silently when one side changes and the other doesn't. The
backend returns a new column; the frontend TS type still says it doesn't
exist; `tsc` compiles, curl works, the user sees zeros. This is the shape of
the Scenario-B bug in `docs/real-world-dogfood/warhammer-40k-calculator.md`.

This skill is the procedural gate that prevents it.

## Iron Law

```
A CONTRACT CHANGE WITHOUT A CLIENT REGEN IS A SILENT BUG.
```

The corollary: if your TaskStep touches a contract source of truth, the same
step MUST regenerate dependents and verify both sides. Splitting the regen
into "the next PR" is the bug.

## Scope — what counts as a contract

- **OpenAPI / Swagger** specs (`openapi.yaml`, `openapi.json`) → generated TS
  client types, Python Pydantic schemas, C# DTOs.
- **GraphQL SDL** (`schema.graphql`) → generated TS types / hooks.
- **Protobuf** (`*.proto`) → generated stubs in every target language.
- **SQL schema / migrations** (`*.sql`, `alembic/versions/*`, `supabase/migrations/*`)
  → ORM models (SQLAlchemy, EF Core, Drizzle) → API response schemas.
- **Shared zod / Pydantic / dataclass schemas** copy-pasted between
  `frontend/` and `backend/` (the worst shape — inline duplication).

A contract is anything where a *shape* is declared in one place and
referenced as a type in another.

## Process

### Phase 1 — Detect

Grep for inline schema duplication — the single loudest smell.

1. For each `interface` / `type` / `BaseModel` / `dataclass` on the frontend,
   grep the backend for the same field set (property names, not types —
   types may differ legitimately between TS and Python).
2. Flag any shape declared in 2+ files where one of those files is a
   "generated" marker (`// @generated`, `# DO NOT EDIT`, `Generated by`) and
   the other is hand-written. Hand-written copies of generated types are
   drift waiting to happen.
3. Flag contracts with **no** regen script documented in `package.json`
   `scripts`, `pyproject.toml` `[tool.*.scripts]`, Makefile, or `justfile` —
   this is a repo gap and must be surfaced to the user before proceeding.

**Evidence collected:** a table of (contract-shape, source-file,
dependent-file, regen-script-present?).

### Phase 2 — Compare

Hash the canonical source vs the last-generated dependent.

1. Compute a canonical hash of the source of truth. For OpenAPI, normalize
   (sort keys, strip whitespace) then `sha256`. For `.proto`, `sha256` the
   file directly. For SQL schema, `pg_dump --schema-only` piped through
   `sha256`.
2. Read the last-known hash from a checked-in marker file (convention:
   `<contract>.hash` alongside the source, or a `contracts.lock.json`
   inventory at repo root). If no marker exists, the repo has no drift
   detection — create the marker as part of this step and flag it.
3. If hashes match: no regen needed; you're done. Skip to verify (Phase 4)
   to confirm both sides still build.
4. If hashes differ: proceed to Phase 3.

### Phase 3 — Regenerate

Run the repo's documented regen script verbatim.

1. Look up the regen command from the project convention (see §Scope). Do
   not invent one.
2. Run it. Capture stdout, stderr, exit code.
3. Update the `.hash` / `contracts.lock.json` marker to the new canonical
   hash.
4. Re-grep for inline duplication (Phase 1 #1) — a regen should have
   eliminated any legitimate hand-written copies. If duplicates remain,
   they are stale manual parallel definitions and must be deleted in this
   step.

### Phase 4 — Verify

Both sides build.

1. Run backend tests: `pnpm test` (TS), `pytest` (Python), `dotnet test`
   (.NET), per repo convention.
2. Run frontend tests + typecheck: `pnpm test` + `tsc --noEmit`, or
   `vitest run` + `tsc --noEmit`.
3. Both exit 0 ⇒ the contract change is consistent. Append the regen
   result + both build exit codes to `.claude/scratchpad.md`.
4. Either side fails ⇒ contract is inconsistent. Do **NOT** claim done.
   Either:
   - The regen produced code that uses a dependency the consumer doesn't
     have yet → update imports in the same step.
   - The regen revealed a breaking change callers don't handle → either
     update callers in the same step, or revert and re-plan the contract
     change to be non-breaking.

## Red flags

| Mental "just X and fix later" | Reality check |
|---|---|
| "I'll update the OpenAPI spec now, regen the TS types next PR." | The TS types are now stale. Every consumer that imports them has a silent bug until "next PR." Regen now. |
| "The frontend TS interface is hand-written, I'll update it manually to match." | You've just created two sources of truth. The regen script is the source of truth; if it doesn't cover this shape, *that's* the fix. |
| "`tsc` passes, ship it." | `tsc` passes because the hand-written type still matches the *old* schema. The runtime bug lives at the JSON boundary where the new field is silently ignored. |
| "The regen script is flaky on Windows; I'll skip it locally." | CI will regen differently. Skipping locally guarantees drift between your commit and CI's artifact. Fix the regen, don't skip. |
| "Only one field changed; I'll patch the generated file inline." | Next regen clobbers your patch. Patch the source of truth; regen. |
| "There's no regen script — I'll write the types by hand since that's how the repo works." | That's the drift generator. Surface the missing regen script as the real fix; don't normalize the bug. |
| "Backend tests pass, that's the contract owner — frontend is downstream so it'll catch up." | Downstream callers break silently. Both sides green is the pass-gate. |
| "The `.hash` file is a nuisance; I'll delete it and rely on git." | Git tells you *something* changed, not whether the *derived artifact* is current. The hash is the derivative check. |

## Enforcement

- The `full-stack-integrator` agent is the primary invoker.
- `pre_tool_use` hook may inject this skill's iron law when an Edit/Write
  targets a recognized contract file (`openapi.yaml`, `*.proto`,
  `supabase/migrations/**`, `schema.graphql`) — see `rules/global/security.md`
  pattern for precedent.
- CI should grep for drift (`sha256 of source` vs `<source>.hash`) and fail
  on mismatch. This closes the human-forgets loop.
